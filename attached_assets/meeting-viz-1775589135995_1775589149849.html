Requirements Traceability Matrix

Technician Workflow — CRA Compliance & Physical Safety Verification

Req ID Requirement Priority Source Status Actions
REQ-001 Physical pump inspection on-site for leakage Must Speaker 1 Accepted
Description
Technician must visually inspect the pump at the installation site to detect any signs of leakage or improper installation before operational use.
Acceptance Criteria
✓ No visible leakage at pump seals or connections. ✓ Pump securely mounted. ✓ Visual inspection documented in site report.
CRA Reference
CRA Art. 10 (Secure Default Configuration) — physical integrity as part of product security lifecycle.
REQ-002 Verify pump is installed correctly per manual specifications Must Speaker 1 Accepted
Description
Technician must confirm that the pump is installed in accordance with the manufacturer's installation guidelines, including orientation, connections, and mechanical alignment.
Acceptance Criteria
✓ Installation matches manual diagrams. ✓ All fasteners tight. ✓ Inlet/outlet orientation correct. ✓ Installation report completed.
CRA Reference
CRA Art. 13 (Resilience & Availability) — proper physical installation ensures operational resilience.
REQ-003 Contact manufacturing dept immediately if leakage or incorrect install Must Speaker 1 Pending
Description
If the technician identifies leakage or incorrect installation during REQ-001/REQ-002, they must immediately escalate to the manufacturing department for remediation.
Acceptance Criteria
✓ Escalation procedure defined. ✓ Manufacturing team SLA for response established. ✓ Escalation logged in service system. ✓ Customer notified of issue status within 2 hours.
CRA Reference
CRA Art. 11 (Vulnerability Handling) — defect identification and remediation process.
REQ-004 Verify pump firmware is current and up-to-date Must Speaker 1 Accepted
Description
Technician must check the pump's current firmware version against the latest available version and confirm it is up-to-date before proceeding with CRA compliance verification.
Acceptance Criteria
✓ Firmware version readable from HMI/controller. ✓ Comparison against known latest version. ✓ If outdated, initiate firmware update workflow. ✓ Result documented.
CRA Reference
CRA Art. 9 (Software Update Mechanism) — essential for vulnerability remediation and security patch deployment.
REQ-005 Review pump maintenance history and last audit control Should Speaker 1 Pending
Description
Technician should review the pump's maintenance log to determine when it was last serviced and when the most recent audit control bus test was performed.
Acceptance Criteria
✓ Maintenance schedule accessible. ✓ Last service date identified. ✓ Last audit control date confirmed. ✓ Any overdue maintenance flagged. ✓ Recommendation for next service issued.
CRA Reference
CRA Art. 12 (Logging & Monitoring) — audit trail and lifecycle tracking for accountability.
REQ-006 Assess pump readiness for CRA compliance update Must Speaker 1 Pending
Description
Technician must verify that the pump system is ready to receive CRA compliance configurations, including evaluation of network connectivity, firmware compatibility, and prerequisites for PIN code provisioning.
Acceptance Criteria
✓ Network connectivity confirmed. ✓ Firmware version supports CRA compliance features. ✓ System prerequisites met (memory, storage). ✓ CRA readiness checklist completed. ✓ Any blockers identified and escalated.
CRA Reference
CRA Art. 10 (Secure Default Configuration) & Art. 5 (Access Control) — foundational for compliance implementation.
REQ-007 Configure PIN code for manufacturing site access Must Speaker 1 Blocked
Description
Technician must configure a secure PIN code on the pump/controller to restrict access at the manufacturing site level, preventing unauthorized configuration changes or tampering.
Acceptance Criteria
✓ PIN code successfully set on device. ✓ PIN format meets security requirements (min 6 chars, alphanumeric). ✓ PIN stored securely (hashed). ✓ Manufacturing personnel briefed on PIN. ✓ Recovery procedure documented.
CRA Reference
CRA Art. 5 (Access Control) — essential security function for role-based access and device hardening.
Blocker
Awaiting digital department confirmation on PIN implementation capability on current controller hardware.
REQ-008 Configure PIN code for customer site access control Must Speaker 1 Blocked
Description
Technician must configure a separate, customer-specific PIN code on the pump/controller to enforce role-based access control at the customer installation site, limiting operational changes to authorized personnel.
Acceptance Criteria
✓ Customer PIN code successfully set. ✓ PIN differs from manufacturing PIN. ✓ PIN communicated securely to customer (NOT via email). ✓ Customer acknowledges receipt and understanding. ✓ PIN change option available to customer after handoff.
CRA Reference
CRA Art. 5 (Access Control) & Art. 10 (Secure Default Configuration) — multi-role access separation.
Blocker
Awaiting digital department confirmation on multi-PIN support and customer notification procedure approval.
REQ-009 Escalate to digital department if PIN code capability unavailable Must Speaker 1 Pending
Description
If the pump/controller does not support PIN code configuration or the feature is unavailable, technician must immediately escalate to the digital department for remediation or alternative access control solution.
Acceptance Criteria
✓ Escalation procedure defined. ✓ Digital team SLA for response (4 hours). ✓ Escalation ticket created with full system context. ✓ Workaround documented if temporary. ✓ Customer notified of delay and ETA.
